# PNN 프론트엔드 라우팅 및 UI 명세서 (Routing & UI Specification)

## 1. 메인 홈 화면 (Index / Home)
- **대상 디자인**: `main-index-page.jpg`
- **파일명**: `HomeScreen.tsx` (가칭)

### 1.1 UI 구성 요소
1. **헤더 (최상단)**
   - PNN 로고 이미지 (추후 추가를 위해 빈 `Image` 태그 또는 View 자리 확보)
   - 서비스 제목 텍스트 배치

2. **메인 검색창**
   - 사각형(Rectangle) 디자인의 입력창
   - Placeholder 텍스트: (디자인과 동일하게 적용 예정)
   - *동작*: 터치 시 '상세검색'(`DrugSearch-page`) 화면으로 이동하거나, 입력된 검색어를 파라미터로 넘기며 이동하는 구조로 추정 (추후 명확히 정의 필요).

3. **핵심 기능 버튼 영역 (4개)**
   - 로고/아이콘 영역 확보 (추후 추가용)
   - 제목과 설명 텍스트는 사진(디자인)과 동일하게 작성.
   - 각 버튼의 레이아웃은 공통 `ActionCard` 컴포넌트로 묶어 재사용.

### 1.2 라우팅 (버튼 클릭 시 이동 플로우)
| 버튼 이름 | 이동할 화면 디자인 | 연결될 컴포넌트(가칭) | 비고 |
| :--- | :--- | :--- | :--- |
| **상세검색** | `DrugSearch-page.jpg` | `DrugSearchScreen` | 일반적인 약품명/성분명 검색 |
| **식별검색** | `PillIdentify-page.jpg` | `PillIdentifyScreen` | 모양, 색상, 각인 등을 통한 낱알 식별 |
| **약X영양제 비교** | `InteractionCheck-page.jpg` | `InteractionCheckScreen` | (Case A) 처방약과 영양제 간 충돌 검사 |
| **영양제 추천** | `Recommendation-page.jpg` | `RecommendationScreen` | (Case B) 기복용 처방약 기반 안전 성분 추천 |

## 2. 약품 검색 화면 (Drug Search)
- **대상 디자인**: `DrugSearch-page.jpg`
- **파일명**: `DrugSearchScreen.tsx`

### 2.1 UI 구성 요소 (공통 및 개별)
1. **공통 헤더 (뒤로가기)**
   - 우상단에 위치한 '뒤로가기(Back)' 버튼.
   - *규칙*: 메인 페이지와 로딩 화면을 제외한 **모든 페이지에 공통으로 적용**됨. (`Header` 컴포넌트로 분리하여 재사용 예정)

2. **타이틀 및 설명**
   - 제목과 설명은 메인 화면(`main-index-page.jpg`)의 버튼에 적혀있던 내용과 동일하게 사용.

3. **검색창 (재사용 컴포넌트)**
   - 메인 화면에서 사용한 사각형 디자인의 검색창 컴포넌트를 재사용 (`SearchBar` 컴포넌트 분리).
   - Placeholder 내용만 해당 페이지의 용도에 맞게 변경하여 적용.

4. **하단 액션 버튼 영역 (초기화 / 검색하기)**
   - 사진과 동일한 배치(보통 좌우 분할 또는 상하단 등) 및 모양 적용.
   - 폼 입력이나 조건 검색 등 다른 페이지에서도 빈번하게 재사용될 예정이므로 공통 컴포넌트화 예정 (`ActionButtons` 등).

### 2.2 동작 (Action)
- **뒤로가기**: 클릭 시 이전 화면(일반적으로 메인 화면)으로 이동.
- **초기화**: 검색창에 입력된 텍스트 및 선택된 검색 조건을 모두 지움 (상태 초기화).
- **검색하기**: 입력된 검색어를 기반으로 검색 결과 화면(`DrugSearchList-page.jpg` 예상)으로 이동하며 API(`GET /api/drugs/search/detail`) 호출용 파라미터를 전달.

## 3. 식별검색 화면 (Pill Identify)
- **대상 디자인**: `PillIdentify-page.jpg`
- **파일명**: `PillIdentifyScreen.tsx`

### 3.1 UI 구성 요소
1. **공통 헤더 & 타이틀**
   - 상단 뒤로가기 버튼(`Header` 컴포넌트 재사용).
   - 제목 및 상세설명은 메인 화면의 해당 버튼 설명과 동일하게 사용.

2. **식별문자 영역**
   - 앞면 / 뒷면 식별문자를 입력받는 2개의 검색창 배치.
   - 사각형 `SearchBar` (또는 `InputBox`) 공통 컴포넌트를 2개 나란히 재사용.

3. **모양 / 색상 / 제형 선택 영역**
   - 사진과 동일하게 그리드(Grid) 또는 가로 슬라이드 형태로 아이콘 및 텍스트 배치.
   - 각 카테고리별로 1개씩만 선택 가능한 구조(Radio Button 형식의 동작을 하는 Custom Button).
   - 선택된 항목은 색상이 반전되거나 테두리가 강조되는 등 시각적 피드백 제공.

5. **하단 액션 버튼 영역 (재사용)**
   - 상세검색과 동일한 `ActionButtons` (초기화 / 검색하기) 컴포넌트 하단에 고정 배치.

### 3.2 동작 (Action)
- **식별마크 선택 창**: 마크 영역 터치 시 마크 리스트 모달 팝업 오픈.
- **초기화**: 입력된 식별문자 및 선택된 마크, 모양, 색상, 제형 상태를 모두 초기 상태(선택 없음)로 리셋.
- **검색하기**: 선택/입력된 모든 식별 파라미터를 모아 검색 결과 화면(`DrugSearchList-page.jpg` 예상)으로 이동하며 API(`GET /api/drugs/search/pillIdentifier`) 호출.

## 4. 검색 결과 리스트 화면 (Drug Search List)
- **대상 디자인**: `DrugSearchList-page.jpg`
- **파일명**: `DrugListScreen.tsx`

### 4.1 UI 구성 요소
1. **공통 헤더 & 타이틀**
   - 상단 뒤로가기 버튼(`Header` 컴포넌트 재사용).
   - 기존 디자인의 검색창 위치에 **'약품 리스트'** 또는 **'검색 결과'** 라는 화면 제목 텍스트 배치. (검색창 컴포넌트는 사용하지 않음)

2. **검색 결과 요약 텍스트**
   - "검색 결과 N건" 텍스트. N은 검색결과 총 개수(`Page` 객체의 `totalElements`)를 반영. N 부분에만 강조 색상 처리.

3. **약품 리스트 아이템 (Card)**
   - 각 약품은 사진과 동일한 형태의 둥근 모서리 네모 박스(`DrugListItemCard` 재사용 컴포넌트)로 출력.
   - **표시 데이터**: `DrugSearchResponseDto`와 완벽히 매핑.
     - 좌측: 약품 이미지 (`itemImage`) - 이미지가 없을 경우를 대비한 대체(Placeholder) 이미지 처리 필요.
     - 우측 상단: 약품명 (`itemName`) - 길 경우 말줄임표(`...`) 처리.
     - 우측 하단: 제조사명 (`entpName`) - 폰트 크기를 작게 하고 회색 톤 처리.

### 4.2 동작 (Action)
- **무한 스크롤(Paging)**: 스크롤을 맨 아래로 내릴 경우 `page` 파라미터를 증가시켜 추가 데이터를 불러옴(FlatList의 `onEndReached` 활용).
- **리스트 아이템 클릭**: 
  - 특정 약품 카드를 터치하면 해당 약의 고유 식별자(`drugId`)를 넘기며 약품 상세 정보 화면(`DrugDetailScreen` 가칭)으로 이동.
  - *참고*: 이 화면은 단순 조회용이므로 별도의 추가 액션 버튼(선택하기 등)은 넣지 않음.

## 5. 약품 상세 정보 화면 (Drug Detail)
- **대상 디자인**: 
  - `DrugSearchDetail(약품정보)-page.jpg`
  - `DrugSearchDetail-MedicationInfoTab(복약정보).jpg`
  - `DrugSearchDetail-PermitInfoTab(허가정보).jpg`
  - `DrugSearchDetail-DUR(금기 및 경고).jpg`
- **파일명**: `DrugDetailScreen.tsx`

### 5.1 UI 구성 요소
1. **공통 헤더 & 타이틀**
   - 상단 뒤로가기 버튼(`Header` 컴포넌트 재사용).
   - 화면 제목: **"약품정보"** (또는 선택한 약품명).

2. **상단 탭 네비게이션 (Tab Bar)**
   - 4개의 탭 메뉴 구성: **[ 약품정보 | 복약정보 | 허가정보 | DUR ]**
   - 현재 선택된 탭은 글자색 강조 및 하단 밑줄(Indicator) 표시.

3. **고정 헤더 영역 (Sticky Header)**
   - 탭 메뉴 바로 아래에 위치하며, 탭을 전환해도 항상 유지되는 공통 정보.
   - 구성: 약품명(`HeaderInfo.itemName`), 제조사명 (DTO 매핑 필요, 통상 `DrugInfo.entpName`).

4. **탭별 컨텐츠 영역 (Tab Content)**
   - 각 탭 터치 시 아래 내용들이 스크롤 영역에 표시됨. 데이터는 모두 `DrugDetailResponseDto` 구조와 1:1 매핑됨.
   
   - **(1) 약품정보 탭 (`DrugInfo`)**: 
     - 상단에 큼직한 약품 이미지 (`itemImageUrl`).
     - 하단에 구분(일반/전문), 판매사, 제조사, 보험코드, 영문 성분명, 주성분 코드, 분류명, ATC코드 등을 `InfoRow` (가칭) 공통 컴포넌트를 재사용하여 세로 리스트 형태로 나열.
   
   - **(2) 복약정보 탭 (`MedicationInfo`)**: 
     - e약은요 기반의 쉬운 설명.
     - "이 약은 어떤 효능이 있나요?" (효능/효과, `efficacy`), "복용 시 주의사항" (`cautionUse`) 등을 둥근 배경색 박스 안에 말머리 기호(Bullet)와 함께 가독성 높게 표시.
   
   - **(3) 허가정보 탭 (`PermitInfo`)**: 
     - 식약처 전문 정보.
     - 상단 요약 2단 그리드: 품목기준코드, 유효기간, 저장방법, 포장단위.
     - 하단 상세 텍스트: 효능효과(`efficacy`), 용법용량(`dosage`), 주의사항(`caution`)을 세로로 길게 표시. (원문이 길 수 있으므로 스크롤 처리).
   
   - **(4) DUR 탭 (`DurInfo`)**: 
     - 상호작용 및 금기 정보. (현재 백엔드 업데이트로 인해 LLM으로 이관되었으나, 화면의 뼈대 UI는 남겨둠).
     - 병용금기 성분 박스(빨간색 배경 경고) 및 DUR 점검 항목 리스트 카드형태로 나열.

### 5.2 동작 (Action)
- **탭 전환**: 상단 탭 터치 시 하단 컨텐츠 영역만 부드럽게 교체됨 (Top Tab Navigator 또는 내부 State 활용).
- **데이터 로딩**: 이전 화면에서 넘겨받은 `drugId`를 기반으로 `GET /api/drugs/{drugId}` API를 호출하여 전체 데이터를 한 번에 가져오고, 탭별로 필요한 데이터만 골라서 렌더링.
