## 3. 모바일 네트워크 에러 트러블슈팅

### 3.1. 증상

- 검색 → 리스트 화면에서 **무한 로딩** 또는 **네트워크 에러**
- 실제 기기(Expo Go)에서 API 호출 실패

### 3.2. 원인

- **기본 baseURL**: `localhost`, `10.0.2.2`는 에뮬레이터/시뮬레이터 전용
- **실제 기기**: `localhost` = 기기 자신, `10.0.2.2` = Android 에뮬레이터 전용 → 백엔드 PC에 연결 불가

### 3.3. 적용한 수정

- **EXPO_PUBLIC_API_URL** 환경 변수 지원 (`api/client.ts`)
- `.env.local`에 `EXPO_PUBLIC_API_URL=http://{PC_IP}:8080` 설정
- `getNetworkErrorMessage()` — 네트워크 에러 시 연결 URL·체크리스트 표시
- `.env.example` — 사용 가이드 추가

### 3.4. 주의사항

- **192.168.56.1**: VirtualBox 호스트 전용 IP → Wi‑Fi와 다른 네트워크, 실제 기기에서 접근 불가
- **192.168.22.54** 등: PC Wi‑Fi IP 사용 (`ipconfig` → 무선 LAN 어댑터 IPv4)
- **.env.local 사용 시**: `EXPO_PUBLIC_API_URL`이 **모든 환경**을 덮어씀 → 에뮬레이터에서 기존에 잘 되던 10.0.2.2가 무시됨
- **해결**: 에뮬레이터 테스트 시 `EXPO_PUBLIC_API_URL` 주석 처리, 실제 기기 테스트 시에만 설정
