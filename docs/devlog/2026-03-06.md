## 발생한 문제 및 해결

| 문제 | 원인 | 해결 |
|------|------|------|
| `ai-server/`가 `backend/` 안에 생성됨 | AI가 경로를 착각 | 수동 이동 후 잘못된 폴더 삭제 |
| `ModuleNotFoundError: No module named 'app'` | 잘못된 디렉토리에서 실행 + 시스템 Python 사용 | `launch.json` 생성하여 cwd와 인터프리터 경로 고정 |
| `python-multipart` 미설치 에러 | FastAPI의 Form 데이터 처리에 필요 | `requirements.txt`에 추가 후 설치 |
| Python `/api/v1/health` 404 | 서버 재시작 없이 코드 변경 반영 안 됨 | 서버 완전 종료 후 재시작 |
| Spring `/api/health/ai` 빈 화면 | `PythonAiClient`에 trailing slash 이슈 + `HealthController` 파일 롤백 | 코드 재확인 및 수정 |
| `psycopg` ImportError (no pq wrapper) | `psycopg` 바이너리 미포함 | `psycopg-binary`로 재설치 |
| 약가마스터 `VARCHAR(255)` 초과 에러 | 한글상품명·약품규격이 255자 초과 | `DrugPriceMaster.java` 컬럼 길이를 500/2000으로 확대, 테이블 재생성 |

## 주요 기술 결정 및 맥락

- **CSV 적재에 Python 사용**: Spring JPA로 30만 건 `save()` 시 성능 문제 → pandas + psycopg Bulk Insert가 빠르고 간편
- **FastAPI 서버 구조**: Spring Boot의 패키지 구조(controller/service/repository)와 유사하게 계층 분리 (api/services/models)
- **Pydantic Settings**: `.env` 파일을 직접 읽는 대신, 타입 검증이 가능한 Settings 클래스로 환경변수 관리
- **VS Code launch.json**: Python CLI 실행의 번거로움 해소 — F5 한 키로 가상환경 + 올바른 디렉토리에서 서버 시작
