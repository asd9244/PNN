## 2. 설계 사항: 기복용 처방약 기반 안전 영양제 추천 (Case B)

## 1. 개요
Case B는 사용자가 현재 복용 중인 **처방약 목록**을 기반으로, 병용 시 부작용이나 대사 간섭 위험이 있는 **금기 영양 성분(Contraindicated Nutrients)**을 사전에 경고하고, 교차 검증된 **안전한 대체 영양 성분(Safe/Recommended Nutrients)**을 제안하는 파이프라인입니다. 
최종적으로 사용자에게 관련 영양제를 바로 탐색할 수 있는 **딥링크(CTA)**를 제공합니다.

---

## 2. 데이터 흐름 (Data Flow)

1. **요청 수신 (Spring Boot)**
   - 클라이언트가 사용자의 `userId` (또는 기복용 약물 `drugId` 배열)를 전송
2. **처방약 성분 추출 (Spring Boot)**
   - `user_drugs` 테이블에서 복용 중인 약품 목록 조회
   - `drugs.main_ingr_eng` 값을 참조하여 약품별 주성분(영문) 추출 (`/` 단위 분리)
3. **금기 성분 색인 (Spring Boot)**
   - 위에서 추출한 약물 주성분을 `interaction_rules` 테이블에 대입 (`WHERE drug_ingredient IN (...)`)
   - 병용 시 부작용이 우려되는 영양제 성분(nutrient) 및 경고 사유(description_kr, action_guide_kr) 수집
4. **AI 서버로 추천 요청 (Spring Boot ➡️ Python FastAPI)**
   - "처방약 목록"과 "색인된 금기 영양 성분 목록"을 묶어서 Python 서버의 `/api/v1/recommendation/safe-nutrients` 엔드포인트로 전송
5. **안전 성분 추론 및 응답 (Python FastAPI ➡️ Spring Boot)**
   - **LLM 프롬프트**: "환자는 A, B 약물을 복용 중이며, C, D 성분은 상호작용 위험이 있음. 이 조건을 회피하면서 환자에게 유익할 수 있는 보편적이고 안전한 영양 성분 2~3가지를 추천하고 이유를 짧게 작성하라."
   - 추천 결과를 JSON 형태로 반환
6. **최종 응답 및 딥링크 조합 (Spring Boot)**
   - 금기 영양 성분(Warning)과 안전 추천 성분(Recommendation)을 합쳐서 클라이언트용 DTO 생성
   - 추천 성분명을 URL 인코딩하여 커머스 딥링크(예: 아이허브 검색결과) 동적 생성

## 3. 엔드포인트 설계

### 3.1 Spring Boot (클라이언트 노출 API)

**`POST /api/v1/recommendations/safe-nutrients`**

- **설명**: 기복용 처방약을 기준으로 안전한 영양 성분 추천 및 금기 성분 경고 반환
- **Request Body (RecommendationRequestDto)**
  ```json
  {
    "userId": 12345,  // 사용자의 기복용 약물 목록을 조회하기 위한 식별자
    // 혹은 클라이언트가 직접 drug 목록을 넘기는 방식도 고려 가능:
    // "drugIds": [1001, 1002]
  }
  ```
- **Response Body (RecommendationResponseDto)**
  ```json
  {
    "warnings": [
      {
        "nutrient": "Yohimbine",
        "reason": "Droxidopa의 진통 효과를 방해할 가능성이 있어 예상치 못한 약효 변형이 발생할 위험이 있습니다.",
        "actionGuide": "Droxidopa 복용 중에는 Yohimbine 영양제 섭취를 피하거나 반드시 전문의와 상담하십시오."
      }
    ],
    "recommendations": [
      {
        "nutrient": "Magnesium",
        "reason": "현재 복용 중인 Droxidopa와 뚜렷한 상호작용이 보고되지 않았으며, 일반적인 신경/근육 기능 유지에 도움을 줄 수 있습니다.",
        "purchaseLink": "https://kr.iherb.com/search?kw=Magnesium"
      },
      {
        "nutrient": "Omega-3",
        "reason": "항염 및 심혈관 건강에 도움을 줄 수 있으며, 현재 처방약 성분들과의 대사 간섭 위험이 낮습니다.",
        "purchaseLink": "https://kr.iherb.com/search?kw=Omega-3"
      }
    ]
  }
  ```

### 3.2 Python AI Server (Spring Boot ↔ Python 내부 API)

**`POST /api/v1/recommendation/analyze-safe`**

- **설명**: 약물 성분과 금기 성분 목록을 받아 안전한 영양 성분 추천
- **Request Body**
  ```json
  {
    "patient_drugs": ["Droxidopa", "Acetaminophen"],
    "contraindicated_nutrients": ["Yohimbine", "Antioxidants", "Vitamin C"]
  }
  ```
- **Response Body**
  ```json
  {
    "recommended_nutrients": [
      {
        "name": "Magnesium",
        "reason_kr": "현재 복용 중인 Droxidopa와 뚜렷한 상호작용이 보고되지 않았으며, 일반적인 신경/근육 기능 유지에 도움을 줄 수 있습니다."
      },
      {
        "name": "Omega-3",
        "reason_kr": "항염 및 심혈관 건강에 도움을 줄 수 있으며, 현재 처방약 성분들과의 대사 간섭 위험이 낮습니다."
      }
    ]
  }
  ```

## 4. 필요한 DTO 상세 (Spring Boot 측)

### 4.1 `RecommendationRequestDto.java`
```java
import jakarta.validation.constraints.NotNull;
import lombok.Getter;
import lombok.Setter;

import java.util.List;

@Getter
@Setter
public class RecommendationRequestDto {
    @NotNull(message = "사용자 식별자(userId) 또는 drugIds 목록이 필요합니다.")
    // 임시로 drugIds를 직접 받는 형태로 가정 (회원 연동 전)
    private List<Long> drugIds; 
}
```

### 4.2 `RecommendationResponseDto.java`
```java
import lombok.Builder;
import lombok.Getter;
import java.util.List;

@Getter
@Builder
public class RecommendationResponseDto {
    private List<WarningItem> warnings;
    private List<RecommendationItem> recommendations;

    @Getter
    @Builder
    public static class WarningItem {
        private String nutrient;
        private String reason;
        private String actionGuide;
    }

    @Getter
    @Builder
    public static class RecommendationItem {
        private String nutrient;
        private String reason;
        private String purchaseLink;
    }
}
```

- [ ] 본 문서(Case B)에 기재된 FastAPI (Python) 추천 로직 구현

1. **Rule-based 필터링 우선 원칙**: 
   - `warnings` (금기 성분) 추출은 전적으로 Spring Boot에서 DB(`interaction_rules`)를 조회하여 수행합니다. LLM이 금기 사항을 지어내지 않도록 완벽히 분리합니다.
2. **보수적 추천 프롬프트 구성 (Python)**:
   - LLM에게 추천을 요구할 때, *"이 성분들이 환자의 질병을 치료해준다"*는 뉘앙스의 단어를 **절대 사용하지 못하도록** 프롬프트를 통제해야 합니다.
   - 추천의 근거는 오직 *"현재 처방약의 대사 경로와 겹치지 않아 부작용 위험이 상대적으로 적다"*는 점에 초점을 맞춰야 합니다.
3. **복수 처방약(Multi-drug) 처리**:
   - 환자가 여러 약을 먹을 경우, 약물 A에는 안전한 영양제가 약물 B에는 금기일 수 있습니다.
   - 따라서 Spring Boot에서 여러 약의 금기 성분을 모두 추출(Union)하여 하나의 거대한 교집합 회피 목록(`contraindicated_nutrients`)을 만든 후 LLM에 넘겨야 합니다.
