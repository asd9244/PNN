# PNN 개발 세션 로그 — 2026-03-12

## 1. 세션 개요

**메인 주제**: 기존 데이터베이스 완전 폐기 및 공공데이터 기반 신규 DB 구축  
**배경**: SUPP.AI 기반 상호작용 규칙의 품질 문제(Actinium 등 잘못된 데이터)로 기존 DB를 초기화하고, 식약처·한국의약품안전관리원 등 공공데이터로 전면 재구축

---

## 2. 기존 DB 폐기 및 재구축 배경

### 2.1 폐기 사유

- **SUPP.AI 데이터 품질 이슈**: 잘못된 상호작용 정보(예: Actinium 등) 포함
- **매칭 사각지대**: 식약처 처방약 성분 1,464개 중 1,165개가 interaction_rules와 미매칭
- **표기 불일치**: 동일 성분이라도 `Vitamin C` vs `Ascorbic acid`, 염기 표기 차이 등으로 단순 문자열 매칭 실패

### 2.2 재구축 방향

- **공공데이터 기반**: 식약처 의약품 낱알식별, 제품허가 상세정보, 주성분, DUR, 병용금기, e약은요 등 공식 출처 데이터로 전면 교체
- **item_seq(품목일련번호) 중심**: 모든 테이블을 품목기준코드로 연결하는 일관된 스키마 설계

---

## 3. 신규 DB 구축 현황

### 3.1 테이블 구조 (6개)


| 테이블                   | 소스                    | 용도                      |
| --------------------- | --------------------- | ----------------------- |
| drugs_master          | 의약품 낱알식별.csv          | 낱알식별(모양·색상·각인), 약 검색    |
| drug_ingredients      | 의약품 상세조회(주성분).csv     | 주성분 정보                  |
| drug_permit_detail    | 의약품 제품허가 상세정보.csv     | 효능효과·용법용량·주의사항 (PDF 파싱) |
| dur_rules             | DUR유형별 성분 현황_*.csv 8종 | 병용금기·임부금기·용량주의 등 DUR 규칙 |
| drug_contraindication | 한국의약품안전관리원_병용금기약물     | 성분명 기준 병용금기             |
| drug_easy_info        | e약은요정보.csv            | 상비약 복약정보                |


### 3.2 DDL 및 적재 스크립트

- **DDL**: `04_dur_rules.sql`, `05_drug_contraindication.sql`, `06_drug_easy_info.sql`
- **적재 스크립트**: `load_dur_rules.py`, `load_drug_contraindication.py`, `load_drug_easy_info.py`
- **run_ddl.py**: psql 미설치 환경에서 Python(psycopg)으로 DDL 일괄 실행

### 3.3 drug_permit_detail PDF 파싱

- efficacy, dosage, caution 컬럼에 PDF URL → 텍스트 추출 후 업데이트
- `extract_pdf_to_detail.py`: 4.6만 건 × 3 PDF, 약 10시간 이상 소요 예상
- 처리 완료된 행은 URL이 텍스트로 대체되어 재선택되지 않음 (멱등성 보장)
